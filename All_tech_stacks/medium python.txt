1. 🔷 What is a Generator in Python?

👉 A generator is a function that returns values one at a time using yield, 
instead of returning all values at once.

✅ Normal Function vs Generator
🔸 Normal Function

def numbers():
    return [1, 2, 3]

print(numbers())

[1, 2, 3]


🔸 Generator Function
def numbers():
    yield 1
    yield 2
    yield 3

gen = numbers()

print(gen)        # generator object
print(next(gen))  # 1
print(next(gen))  # 2
print(next(gen))  # 3


| Feature   | Normal Function | Generator         |
| --------- | --------------- | ----------------- |
| Keyword   | `return`        | `yield`           |
| Memory    | High            | Low               |
| Execution | All at once     | One by one        |
| Use case  | Small data      | Large/stream data |



A generator in Python is a special type of iterator that yields values lazily using the yield keyword, allowing efficient memory usage and handling of large datasets.

3. Defaultdict & ordereddict 

🔷 1. defaultdict in Python
🧠 Definition

👉 defaultdict is a dictionary that provides a default value for a missing key instead of throwing an error.

✅ Using defaultdict
from collections import defaultdict

d = defaultdict(int)   # default value = 0

d["a"] += 1
d["a"] += 1

print(d)   # {'a': 2}

✔ No error
✔ Automatically initializes missing key


🔷 2. OrderedDict in Python
🧠 Definition

👉 OrderedDict is a dictionary that maintains the insertion order of keys.

from collections import OrderedDict



from collections import OrderedDict

d = OrderedDict()

d["a"] = 1
d["b"] = 2
d["c"] = 3

print(d)

OrderedDict([('a', 1), ('b', 2), ('c', 3)])

| Feature        | `defaultdict`                   | `OrderedDict`         |
| -------------- | ------------------------------- | --------------------- |
| Purpose        | Default values for missing keys | Maintain order        |
| Error handling | Avoids KeyError                 | No impact             |
| Use case       | Counting, grouping              | Caching, ordered data |


defaultdict automatically initializes missing keys with a default value,
while OrderedDict maintains insertion order and provides additional ordering operations like moving elements.

5. Copy, deep copy, and shallow copy of objects in Python

🧠 1. Basic Copy (Assignment)
👉 Definition

👉 This is not a real copy, it just creates a reference to the same object.

a = [1, 2, 3]
b = a

b.append(4)

print(a)  # [1, 2, 3, 4]
print(b)  # [1, 2, 3, 4]


🔷 2. Shallow Copy
👉 Definition

👉 Shallow copy creates a new object, but nested objects are still shared (referenced).

import copy

a = [[1, 2], [3, 4]]
b = copy.copy(a)

b[0].append(99)

print(a)  # [[1, 2, 99], [3, 4]]
print(b)  # [[1, 2, 99], [3, 4]]


🔷 3. Deep Copy
👉 Definition

👉 Deep copy creates a completely independent copy (including nested objects).

import copy

a = [[1, 2], [3, 4]]
b = copy.deepcopy(a)

b[0].append(99)

print(a)  # [[1, 2], [3, 4]]
print(b)  # [[1, 2, 99], [3, 4]]

✔ Everything is copied
✔ No shared references

Shallow copy creates a new object but shares references of nested objects, 
whereas deep copy creates a completely independent copy of the object and all its nested elements.


6. How to write a custom context manager, give a use case 

🔷 What is a Context Manager in Python?
🧠 Definition

👉 A context manager is a construct that allows you to allocate and release resources properly using the with statement.

✔ It ensures:

Setup is done before execution
Cleanup is done after execution (even if errors occur)
