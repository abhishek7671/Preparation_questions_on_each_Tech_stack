1. 🔷 What is a Generator in Python?

👉 A generator is a function that returns values one at a time using yield, 
instead of returning all values at once.

✅ Normal Function vs Generator
🔸 Normal Function

def numbers():
    return [1, 2, 3]

print(numbers())

[1, 2, 3]


🔸 Generator Function
def numbers():
    yield 1
    yield 2
    yield 3

gen = numbers()

print(gen)        # generator object
print(next(gen))  # 1
print(next(gen))  # 2
print(next(gen))  # 3


| Feature   | Normal Function | Generator         |
| --------- | --------------- | ----------------- |
| Keyword   | `return`        | `yield`           |
| Memory    | High            | Low               |
| Execution | All at once     | One by one        |
| Use case  | Small data      | Large/stream data |



A generator in Python is a special type of iterator that yields values lazily using the yield keyword, allowing efficient memory usage and handling of large datasets.

5. Copy, deep copy, and shallow copy of objects in Python

🧠 1. Basic Copy (Assignment)
👉 Definition

👉 This is not a real copy, it just creates a reference to the same object.

a = [1, 2, 3]
b = a

b.append(4)

print(a)  # [1, 2, 3, 4]
print(b)  # [1, 2, 3, 4]


🔷 2. Shallow Copy
👉 Definition

👉 Shallow copy creates a new object, but nested objects are still shared (referenced).

import copy

a = [[1, 2], [3, 4]]
b = copy.copy(a)

b[0].append(99)

print(a)  # [[1, 2, 99], [3, 4]]
print(b)  # [[1, 2, 99], [3, 4]]


🔷 3. Deep Copy
👉 Definition

👉 Deep copy creates a completely independent copy (including nested objects).

import copy

a = [[1, 2], [3, 4]]
b = copy.deepcopy(a)

b[0].append(99)

print(a)  # [[1, 2], [3, 4]]
print(b)  # [[1, 2, 99], [3, 4]]

✔ Everything is copied
✔ No shared references

Shallow copy creates a new object but shares references of nested objects, 
whereas deep copy creates a completely independent copy of the object and all its nested elements.

7. Monkey Patching in Python

✅ Definition

Monkey patching means modifying or extending code at runtime (dynamically) without changing the original source code.

You can:

Replace methods
Add new methods/attributes
Modify behavior of libraries/classes


🧠 Why used (Real-time use cases)

class Employee:
    def get_salary(self):
        return 50000

8. Implement Queue, Stack & Linked List in Python
🔹 A. Stack (LIFO – Last In First Out)
✅ Definition

Last inserted element is removed first.

🧪 Real-time example
Undo/Redo operations
Browser back button



class Stack:
    def __init__(self):
        self.stack = []

    def push(self, item):
        self.stack.append(item)

    def pop(self):
        if not self.is_empty():
            return self.stack.pop()
        return "Stack is empty"

    def peek(self):
        return self.stack[-1] if self.stack else None

    def is_empty(self):
        return len(self.stack) == 0

# Usage
s = Stack()
s.push(10)
s.push(20)
print(s.pop())   # 20


B. Queue (FIFO – First In First Out)
✅ Definition

First inserted element is removed first.

🧪 Real-time example
Ticket booking system
Task scheduling


from collections import deque

class Queue:
    def __init__(self):
        self.queue = deque()

    def enqueue(self, item):
        self.queue.append(item)

    def dequeue(self):
        if self.queue:
            return self.queue.popleft()
        return "Queue is empty"

    def peek(self):
        return self.queue[0] if self.queue else None

# Usage
q = Queue()
q.enqueue(1)
q.enqueue(2)
print(q.dequeue())   # 1


C. Linked List
✅ Definition

A linear data structure where each element (node) contains:

data
reference (pointer) to next node
🧪 Real-time example
Memory-efficient data storage
Implementing stacks/queues internally
Navigation systems


class Node:
    def __init__(self, data):
        self.data = data
        self.next = None


class LinkedList:
    def __init__(self):
        self.head = None

    def insert(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        
        temp = self.head
        while temp.next:
            temp = temp.next
        temp.next = new_node

    def display(self):
        temp = self.head
        while temp:
            print(temp.data, end=" -> ")
            temp = temp.next
        print("None")

# Usage
ll = LinkedList()
ll.insert(10)
ll.insert(20)
ll.insert(30)
ll.display()   # 10 -> 20 -> 30 -> None

11. Singleton Class (Restrict Object Creation)
✅ Definition

A Singleton ensures that:

Only one instance of a class is created throughout the program.

🧠 Real-time Use Case
Database connection
Logging system
Configuration manager


class Singleton:
    _instance = None

    def __new__(cls):
        if cls._instance is None:
            print("Creating new instance")
            cls._instance = super().__new__(cls)
        return cls._instance

# Usage

12. What are Iterators in Python?
✅ Definition

An iterator is an object that:

Implements __iter__() and __next__()
Allows sequential access to elements

🧠 Real-time Use Case
Reading large files line by line
Streaming data processing
Generators

nums = [1, 2, 3]

it = iter(nums)

print(next(it))  # 1
print(next(it))  # 2
print(next(it))  # 3


class Counter:
    def __init__(self, max):
        self.max = max
        self.current = 0

    def __iter__(self):
        return self

    def __next__(self):
        if self.current < self.max:
            self.current += 1
            return self.current
        else:
            raise StopIteration

# Usage
c = Counter(3)
for num in c:
    print(num)

16. Lazy Functions for Reading Large Files
✅ Definition

Lazy functions process data chunk by chunk instead of loading everything into memory.

🧠 Real-time Use Case
Log processing (GB files)
ETL pipelines
Data engineering

def read_large_file(file_path):
    with open(file_path, 'r') as f:
        for line in f:
            yield line.strip()   # lazy yield

# Usage
for line in read_large_file("bigfile.txt"):
    print(line)

class FileManager:
    def __enter__(self):
        print("Opening file")
        self.file = open("test.txt", "w")
        return self.file
