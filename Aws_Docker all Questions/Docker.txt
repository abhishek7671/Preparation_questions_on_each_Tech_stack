1. What is Docker?
✅ Definition

Docker is a containerization platform that allows you to package an application along with its dependencies into a standardized unit called a container, which can run anywhere.

docker --version        # Check Docker installation
docker info             # View Docker system info



📦 2. What are Docker Containers?
✅ Definition

A Docker container is a lightweight, portable runtime environment that contains:

Application code
Libraries
Dependencies

👉 Containers run on top of the host OS (no full OS needed).


docker run nginx              # Run a container
docker ps                     # List running containers
docker stop <container_id>    # Stop container
docker rm <container_id>      # Remove container



⚔️ 3. Docker vs Virtual Machines (VMs)
✅ Key Differences

| Feature      | Docker (Containers) | Virtual Machines |
| ------------ | ------------------- | ---------------- |
| OS           | Shares host OS      | Full OS per VM   |
| Size         | Lightweight         | Heavy            |
| Startup Time | Seconds             | Minutes          |
| Performance  | Faster              | Slower           |
| Resource Use | Low                 | High             |


💡 Real-Time Example
VM → Install Ubuntu OS → Install Python → Run app
Docker → Just run container → app starts instantly



🌐 4. What is Docker Hub?
✅ Definition

Docker Hub is a cloud-based registry where you can:

Store Docker images
Share images
Download public images

👉 Like GitHub, but for Docker images.


docker pull nginx            # Download image
docker push username/image   # Upload image
docker search redis          # Search images




🏗️ 5. Docker Architecture
✅ Components
Docker Client
CLI (docker commands)
Docker Daemon
Runs containers & manages images
Docker Images
Blueprint/template
Docker Containers
Running instances of images
Docker Registry (Docker Hub)
Stores images



🧱 6. What is a Docker Image?
✅ Definition

A Docker image is a read-only template used to create containers.

👉 It contains:

Application code
Dependencies
Environment setup
💡 Real-Time Example

Image = Blueprint
Container = Running instance

Like:

Image → Class
Container → Object


docker images               # List images
docker pull node            # Download image
docker rmi <image_id>       # Remove image




7. Difference: docker run vs docker start
✅ docker run

👉 Creates + starts a container

docker run nginx


✅ docker start

👉 Starts an existing stopped container

docker start <container_id>




8. Main Components of a Docker Container
✅ Definition

A Docker container is built using several internal components:

🔑 Core Components:
Docker Image → Blueprint/template
Application Code → Your app (Django, Node, etc.)
Dependencies → Libraries, packages
Runtime → Environment to execute code
Filesystem → Layered storage system
Network Interfaces → Communication
Volumes (optional) → Persistent storage
💡 Real-Time Example

A Django app container includes:

Python runtime
Django code
Requirements.txt dependencies
SQLite/Postgres connection


docker run -d -p 8000:8000 my-django-app




9. What is the Purpose of a Dockerfile?
✅ Definition

A Dockerfile is a script with instructions to build a Docker image.

👉 It automates environment setup.

💡 Real-Time Example

Instead of manually installing Python, pip, dependencies →
You define everything once in a Dockerfile.


FROM python:3.10
WORKDIR /app
COPY . .
RUN pip install -r requirements.txt
CMD ["python", "app.py"]



docker build -t myapp .
docker run myapp

11. What are Docker Volumes?
✅ Definition

Docker volumes are used for persistent data storage outside containers.

👉 Containers are temporary, volumes store permanent data.

💡 Real-Time Example

MySQL container:

Without volume → data lost after container removal
With volume → data persists


docker volume create myvolume
docker run -d -v myvolume:/data mysql

14. What is Docker Compose?
✅ Definition

Docker Compose is a tool to define and run multi-container applications using a YAML file.

💡 Real-Time Example

Django backend

15. Difference between docker pull and docker build
✅ docker pull (Definition)

Downloads an image from a remote registry like Docker Hub.

✅ docker build (Definition)

Creates a new Docker image from a Dockerfile.



| Feature  | docker pull     | docker build        |
| -------- | --------------- | ------------------- |
| Source   | Docker Hub      | Local Dockerfile    |
| Purpose  | Download image  | Create custom image |
| Use Case | Ready-made apps | Custom apps         |



docker pull nginx

docker build -t myapp .

23. What is a Multi-Stage Build in Docker?
✅ Definition

A multi-stage build uses multiple FROM statements in a Dockerfile to reduce image size.

👉 Keeps only required files in final image.

💡 Real-Time Example

Build a Python app:

Stage 1 → install dependencies
Stage 2 → copy only final code



# Stage 1 (build)
FROM python:3.10 as builder
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt

# Stage 2 (final)
FROM python:3.10-slim
WORKDIR /app
COPY --from=builder /usr/local/lib/python3.10 /usr/local/lib/python3.10
COPY . .
CMD ["python", "app.py"]



24. How do you handle Environment Variables in Docker?
✅ Definition

Environment variables are used to configure applications dynamically without changing code.

💡 Real-Time Example

Store DB credentials:

DB_HOST
DB_USER
DB_PASSWORD

🧪 Methods
1. Using -e
docker run -e DB_HOST=localhost -e DB_USER=admin myapp
2. Using .env file
docker run --env-file .env myapp
3. In Dockerfile
ENV APP_ENV=production



25. What is the purpose of Docker --link flag?
✅ Definition

--link was used to connect containers and share environment variables.

👉 ⚠️ Now deprecated → replaced by Docker networks


docker network create mynet
docker run -d --network mynet --name db mysql
docker run -d --network mynet myapp

27. How do you debug issues inside a Docker container?
✅ Definition

Debugging involves inspecting logs, accessing container shell, and checking processes.

💡 Real-Time Example

Your Django app is not starting → debug inside container.

🧪 Commands
1. Check logs
docker logs <container_id>
2. Enter container
docker exec -it <container_id> bash
3. Check running processes
ps aux
4. Inspect container
docker inspect <container_id>
5. Check resource usage
docker stats
